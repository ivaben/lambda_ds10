- What, in your opinion, is an important part of code reviews? That is, what is
  something you pay attention to when you review code, and that you appreciate
  when others do the same for your code?

Code review is a very fundamental part of writing code. 
It helps to identify defects, bugs, and simple errors. 
The writer should really appreciates that some other developers examine source code and provide feedback to the authors. 
The goal of code review should be thatthe person who is not involved in the project will be able 
to understand what it does and to see preconditions of its use. 


- We have an awful lot of computers here, and it gets pretty confusing with
  slightly different things running on all of them. How could containers help us
  improve this situation?

  In this situation, there are benefits to using containers. 
  They quick spinning up apps and reduce security updates. 
  It also means less code to transfer, migrate and upload workloads.  
  They are lightweight and containers have a native performance and all containers share the host OS.
